我一直有種研究型的習慣——做每件事之前，總忍不住先花時間找最好的方法。

比如說，光是找個適合的筆記軟體，我就試了 Notion、Anytype、Logseq、OneNote、Evernote等等，每個都用上一段時間，才選定 Obsidian。喜歡它的介面直覺，檔案之間能做連結，更重要的是能直接用 Claude Code 整理筆記。

這種研究模式幾乎成了我做所有事的縮影：學程式、看天文、研究加密貨幣、搞 AI 工作流等等， 都是先埋進資料裡，比較、測試，最後整理出自己的做法。但時間久了，發現一個問題：這些花了好多時間研究的東西，常常過一陣子就忘。筆記零零散散，回頭看也未必看得懂。如果有些理解是錯的，我也沒機會知道，所以才一直想寫部落格，但總覺得把筆記整理成文章太花時間。那些零散的筆記，要變成別人能讀的東西，感覺要再花費額外大量的時間。

直到最近開始用 Claude Code 搭配 Obsidian，發現它可以幫我把散亂的筆記快速整理成有架構的內容，最大的門檻突然變得沒那麼麻煩，所以就決定開始寫了。一方面是想逼自己更認真學習，一方面也是希望如果有人看到，發現哪裡有問題或歡迎批評指教。也希望能遇到對類似主題志同道合的人。

謝謝你讀到這裡，這是我的第一篇文章。  
之後會慢慢分享一些學習和研究的筆記，往後會陸續分享我的研究兼學習心得。
